Log Viewer Application Flow

1. Deployments Dashboard /

The entry point of the application. Users see a high-level list of all deployments.

Key Features: Search by ID/Title, Status Badges (Success/Failed/Running), Creation timestamps, and direct links to details.
2. Deployment Detail View /deployments/[id]

Detailed timeline for a specific deployment run.

Key Features: Vertical timeline of steps, Step status visualization, Duration tracking, and navigation to specific step logs.
3. Step & Log Analysis /steps/[id]

Granular view of a single step within a deployment.

Key Features: Lifecycle event tracking (Start/Finish), Log summary (Stdout vs Stderr count), and a full Log Panel with ANSI-supported output rendering.
Alternative: Task Runner /tasks

A specialized interface to manually trigger new deployment tasks.

Inputs: App Name, Working Directory, and a JSON-formatted list of shell commands to execute as steps.
