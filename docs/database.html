Database Schema Documentation

The Log Viewer uses a MySQL database managed via Prisma ORM. This document describes the tables, columns, and relationships.

Tables

deployments (Top-level deployment runs)
Column Type Description
id CHAR(36) Primary Key (UUID).
title VARCHAR(100) Human-readable title (e.g., "moneygoup"). Defaults to "myapp".
created_at DATETIME Timestamp when the deployment record was created.
steps (Individual tasks within a deployment)
Column Type Description
id INT (AI) Primary Key. Auto-incrementing integer.
deployment_id CHAR(36) Foreign Key referencing deployments.id.
name VARCHAR(255) Name of the step (e.g., "docker_build").
status ENUM Current status (see steps_status enum).
started_at DATETIME When the step execution started.
finished_at DATETIME When the step execution completed.
duration_ms INT Total duration in milliseconds.
created_at DATETIME Record creation timestamp.
events (Lifecycle milestones for a step)
Column Type Description
id INT (AI) Primary Key.
step_id INT Foreign Key referencing steps.id.
event_type ENUM Type of event (see events_event_type enum).
created_at DATETIME Timestamp of the event.
child_events (Detailed logs/output for a step)
Column Type Description
id INT (AI) Primary Key.
step_id INT Foreign Key referencing steps.id.
log TEXT The actual log message content.
log_type ENUM Standard output or error (see log_type enum).
created_at DATETIME When this log entry was captured.

Relationships

  • Deployments → Steps: One-to-Many. One deployment contains multiple steps.
  • Steps → Events: One-to-Many. Each step has multiple lifecycle events (start, finish, etc).
  • Steps → Logs: One-to-Many. Each step can produce many log lines (child_events).

Enums

steps_status
  • pending
  • running
  • success
  • failed
events_event_type
  • start
  • finish
  • error
child_events_log_type
  • stdout
  • stderr
